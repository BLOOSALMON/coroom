# PRD: coroom (회의실 예약 시스템)

## 1. 개요

| 항목 | 내용 |
|---|---|
| 앱 이름 | coroom |
| 한 줄 정의 | 사내 회의실(1~6번)의 예약 현황을 한눈에 보고, 빈 시간대를 클릭해 바로 예약할 수 있는 웹 기반 회의실 예약 시스템 |
| 데이터베이스 | Supabase (PostgreSQL) |
| 기존 자산 | 방금 만든 PRD.md대로 회의실 예약 시스템 만들어줘`meetingroom_reservations.xlsx` (회의실목록/예약현황 시트) — 초기 데이터 마이그레이션 소스 |

### 배경 및 문제 정의
기존에는 엑셀(`meetingroom_reservations.xlsx`)로 회의실 예약을 관리해왔다. 이 방식은 다음 문제가 있다.
- 실시간 동시 편집이 어렵고, 중복 예약(더블부킹)이 발생해도 즉시 확인되지 않음
- 예약 현황을 한눈에 보기 어렵고, 빈 시간을 찾으려면 셀을 일일이 확인해야 함
- 예약/취소 이력 추적이 번거로움

coroom은 이 엑셀 워크플로우를 웹 대시보드로 대체하여, 회의실 6개의 예약 현황을 시간대별 보드로 시각화하고 빈 칸을 클릭하는 것만으로 예약이 완료되도록 한다.

## 2. 목표

- **핵심 목표**: 예약 현황 보드에서 빈 시간대 클릭 → 예약 완료까지 3단계 이내
- 회의실별/일자별 예약 현황을 실시간으로 확인 가능
- 중복 예약(동일 회의실, 겹치는 시간) 방지
- 기존 엑셀 데이터(회의실 6개, 예약 이력)를 초기 데이터로 이관

## 3. 대상 사용자

- **임직원(일반 사용자)**: 회의실 조회, 예약 생성/취소, 본인 예약 확인
- (확장 여지) **관리자**: 회의실 정보(장비, 비고 등) 관리, 전체 예약 관리 — MVP 범위 외, 5장 참고

## 4. 핵심 기능

### 4.1 예약 현황 보드 (메인 화면)
- 세로축: 시간 (예: 09:00~19:00, 30분 또는 1시간 단위)
- 가로축: 회의실 1~6번
- 날짜 선택(기본값: 오늘), 이전/다음 날짜 이동
- 각 셀은 상태에 따라 다르게 표시
  - **빈 시간대**: 클릭 가능, 호버 시 "예약하기" 표시
  - **예약됨(확정)**: 예약자/회의 제목 표시, 클릭 시 상세 보기
  - **취소된 예약**: 보드에는 빈 시간대로 노출(취소 이력은 별도 조회에서만 확인)
- 회의실 헤더에 회의실명, 수용인원, 보유장비 요약 표시(툴팁 또는 인라인)

### 4.2 빈 시간대 클릭 예약
- 보드에서 빈 셀 클릭 → 예약 모달 오픈
  - 선택된 회의실/날짜/시작시간 자동 입력
  - 종료시간 선택(드래그로 여러 셀 선택 시 자동 범위 지정도 고려)
  - 입력 항목: 예약자명, 부서, 회의 제목
- 제출 시 서버(Supabase)에서 동일 회의실·시간대 중복 여부 검증 후 저장
- 저장 성공 시 보드에 즉시 반영

### 4.3 예약 관리
- **내 예약 목록**: 예약자 기준 필터 (이름 또는 추후 로그인 계정 기준)
- **예약 취소**: 목록/모달에서 취소 버튼 → 상태를 '취소'로 변경(하드 삭제 아님, 이력 보존)
- **예약 상세 보기**: 회의실, 일시, 예약자, 부서, 제목, 상태

### 4.4 회의실 정보
- 회의실 1~6번 목록: 이름, 수용인원, 층, 보유장비, 비고
- 예: 5번 회의실(대회의실)은 "임원 보고용 우선 배정" 같은 비고를 보드/모달에 안내 문구로 노출

## 5. 화면 구성 (UI/UX)

1. **메인 보드 화면**
   - 상단: 날짜 선택기, 오늘로 이동 버튼
   - 본문: 회의실(열) x 시간(행) 그리드
   - 범례: 예약가능/예약됨 색상 구분
2. **예약 모달** (빈 셀 클릭 시)
   - 회의실/날짜/시간 (사전 입력, 시간은 조정 가능)
   - 예약자명 / 부서 / 회의 제목 입력
   - "예약하기" 버튼
3. **예약 상세 모달** (예약된 셀 클릭 시)
   - 예약 정보 표시 + "취소하기" 버튼(본인 예약인 경우)
4. **내 예약 목록 페이지**
   - 예약자명으로 필터한 리스트, 상태(확정/취소) 표시

## 6. 데이터 모델 (Supabase)

기존 `meetingroom_reservations.xlsx`의 두 시트를 기준으로 스키마를 설계한다.

### `rooms` (회의실목록 시트 대응)
| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | int (PK) | 회의실번호 (1~6) |
| name | text | 회의실명 (예: "1번 회의실 (소회의실 A)") |
| capacity | int | 수용인원 |
| floor | text | 층 |
| equipment | text | 보유장비 |
| notes | text | 비고 |

### `reservations` (예약현황 시트 대응)
| 컬럼 | 타입 | 설명 |
|---|---|---|
| id | uuid (PK, default gen_random_uuid()) | 예약 고유 ID (엑셀의 "B2026XXX" 형식 예약번호는 표시용 별도 컬럼으로 유지 가능) |
| room_id | int (FK → rooms.id) | 예약된 회의실 |
| reserver_name | text | 예약자 |
| department | text | 부서 |
| title | text | 회의 제목 |
| reservation_date | date | 예약일자 |
| start_time | time | 시작시간 |
| end_time | time | 종료시간 |
| status | text (enum: '확정', '취소') | 예약 상태 |
| created_at | timestamptz (default now()) | 생성 시각 |

### 제약사항
- 동일 `room_id` + `reservation_date`에서 `status = '확정'`인 예약끼리 시간이 겹치면 안 됨 (애플리케이션 레벨 검증 + DB 트리거/제약 고려)
- `end_time` > `start_time`

## 7. 비기능 요구사항

- 보드 로딩은 선택한 날짜의 예약만 조회하여 빠르게 렌더링
- 동시에 여러 사용자가 같은 빈 시간대를 클릭할 수 있으므로, 저장 시점에 서버 측 중복 검증 필수(레이스 컨디션 방지)
- 모바일/데스크톱 반응형 대응(그리드가 좁은 화면에서도 스크롤 가능하도록)

## 8. MVP 범위 밖 (향후 확장 고려)

- 로그인/인증 연동 (현재는 이름 직접 입력 방식)
- 관리자 승인 절차, 회의실 정보 관리 화면
- 예약 알림(이메일/슬랙), 반복 예약
- 회의실 이용률 통계/리포트

## 9. 마이그레이션 계획

1. `rooms` 테이블에 `meetingroom_reservations.xlsx`의 "회의실목록" 시트 6개 행 삽입
2. `reservations` 테이블에 "예약현황" 시트의 예약 이력(현재 26건) 삽입, 상태값 그대로(확정/취소) 매핑
